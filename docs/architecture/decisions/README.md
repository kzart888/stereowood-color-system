# Architecture Decision Log

This folder stores decision records for the architecture-upgrade phase.

## Naming
- `A0-0001-*.md`, `A1-0001-*.md`, etc.
- Prefix matches batch in `docs/architecture/PHASE_A_BLUEPRINT.md`.

## Required Sections
1. Context
2. Decision
3. Alternatives considered
4. Consequences
5. Rollback strategy
6. Validation evidence

## Rule
- Each Phase A batch must add or update at least one decision record.
